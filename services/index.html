BottleServiceRunner and TopShelf

You can enable rake helpers for a project hosted by BottleServiceRunner and TopShelf v3.0 with the snippet below:

@solution = FubuRake::Solution.new do |sln|

end

FubuRake::BottleServices.new({
  # Use this option to use a different prefix for the 
  # generated tasks
  :prefix => "service", 

  # This option needs to point at the directory of the build
  # products of the service project
  :dir => "src/DiagnosticsHarness/bin/#{@solution.compilemode}", 
  :name => 'ft-harness', 
  :instance => 'something',
  :user => 'user name',
  :password => 'password',
  :autostart => true,
  :manual => true,
  :disabled => true,
  :delayed => true,
  :local_service => true,
  :network_service => true,
  :interactive => true,
  :description => 'something'
  :manual => true
})

Which will give you these rake tasks, assuming the default prefix 'service':

Task Description
service:clean Deletes any exploded bottle content in the directory
service:console Starts a new command window and runs the service in a console
service:install Installs the service locally
service:start Attempts to start the named service if it is installed
service:stop Attempts to stop the installed service
service:uninstall Uninstalls the named service

See TopShelf Command Line reference for more description about the rake task options and how they relate to Windows service installation."

You will need to install the BottleServiceRunner nuget on the project so that the BottleServiceRunner.exe is in the path of the project.
