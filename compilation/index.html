Compilation

FubuRake supports compilation with both MSBuild on Windows and xBuild from Mono with Linux. FubuRake's compilation support will detect the platform for you and switch the compilation strategy accordingly.

The Default :compile Task

Setting up the :compile task is generally declarative. If you only have one *.sln file in the code tree, FubuRake will find that automatically and use the default compilation options to create the :compile task. If you do have multiple solution files or just prefer the build file to be more explicit, you only need to set the :solutionfile attribute as in the code below: 
solution = FubuRake::Solution.new do |sln|
	sln.compile = {
		:solutionfile => 'src/FubuRake.sln'
	}
end
There are other compilation options explained in the next section.

Compilation Options

Typically you will just use the FubuRake options and only specify the :solutionfile, but it's still good to know what the levers are that you could pull to customize the compilation.
Key Description
:compilemode The compilation mode. Uses the :compilemode from the FubuRake::Solution.options by default, but can be overriden.
:solutionfile The path to the *.sln or *.csproj file to compile
:clrversion The CLR version of MSBuild or xBuild. Uses :clrversion from the FubuRake::Solution.options by default, but can be overriden

Other Compilation Steps

If you need additional compilation tasks beyond the default :default task, you can use the :compile_step method to generate additional compilation tasks using the same options as the rest of the solution like so: 
@solution = FubuRake::Solution.new do |sln|
	sln.compile = {
		:solutionfile => 'src/Bottles.sln'
	}
				 
	sln.compile_step :compile_console, 'src/Bottles.Console/Bottles.Console.csproj'
	sln.compile_step :compile_bottle_project, 'bottles-staging/BottleProject.csproj'
end
The first argument (:compile_console) is the name of the task and the second argument is the relative path to either a *.csproj or *.sln file.

Adding Prerequisite Tasks

If you need to chain other custom tasks as dependencies to the :default task, use the precompile property to add an array of task names as in the following example: 
@solution = FubuRake::Solution.new do |sln|
	sln.compile = {
		:solutionfile => 'src/Bottles.sln'
	}

	sln.precompile = [:bottle_assembly_package]
end

desc "does the assembly bottling of the AssemblyPackage test project"
task :bottle_assembly_package => [:compile_bottle_project] do
  # do the bottling...
end
